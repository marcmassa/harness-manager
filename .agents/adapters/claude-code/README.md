# claude-code adapter

This adapter renders the canonical manifest `.agents/agentic.json` into
Claude Code's native config format:

- `./CLAUDE.md` — top-level project instructions (auto-loaded by Claude Code).
- `./.claude/agents/<name>.md` — one file per sub-agent.
- `./.claude/commands/<name>.md` — one file per slash command.
- `./.claude/settings.json` — file-level permissions and project settings.

## How it is rendered

`./.agents/bootstrap.sh claude-code` (or equivalently
`./.agents/adapters/_common/render.py --cli claude-code --root .`) reads
`.agents/agentic.json`, applies `project_detect` rules, and writes all four
file groups. No LLM is involved.

## Output paths

- `./CLAUDE.md` (gitignored)
- `./.claude/agents/<name>.md` for each sub-agent (gitignored)
- `./.claude/commands/<name>.md` for each command (gitignored)
- `./.claude/settings.json` (gitignored)

To verify all generated files are in sync with the manifest, run
`./.agents/bootstrap.sh claude-code --check` or `./.agents/bootstrap.sh --check`.

## Field mapping (`.agents/agentic.json` → Claude Code)

| `agentic.json` | Claude Code output | Notes |
|---|---|---|
| `instructions: [...]` | top of `CLAUDE.md` (after the framework header) | Listed as references for the model to read |
| `subagents[*]` | one `.claude/agents/<name>.md` with frontmatter `description` | Body is a stub pointing at `.agents/subagents/<name>/SUBAGENT.md` |
| `subagents[*].permission` | (logged in the per-agent stub) | **Note**: Claude Code's per-sub-agent file permissions are configured differently than opencode's inline `permission` block. The current adapter records the contract in prose. If you need hard file-level enforcement at the sub-agent level, configure `permissions.allow`/`deny` in `.claude/settings.json` (currently scoped at the project level, not per sub-agent) |
| `commands[*]` | one `.claude/commands/<name>.md` with frontmatter `description` | Body is a stub pointing at `.agents/commands/<name>.md` |
| `project_detect[*]` | "Detected Stack" section in `CLAUDE.md` | Surfaced as a list of stack labels |
| `project_detect[*].apply.implementer_permission_overrides` | merged into the `implementer` permission contract | Documented in prose |

## Caveats and limitations

- **Sub-agent permissions are project-wide**: Claude Code's `settings.json`
  applies to the project as a whole, not per sub-agent. The per-sub-agent
  contracts in `.claude/agents/<name>.md` are documentation; the actual
  enforcement happens via `.claude/settings.json` at the project level.
- **Skills discovery**: Claude Code auto-discovers `**/SKILL.md` under
  `.claude/skills/`. The adapter does not generate this directory; symlink
  or copy `.agents/skills/` to `.claude/skills/` (one-time setup):
  ```bash
  ln -s ../.agents/skills .claude/skills
  ```
- **Slash commands are stubs**: each `.claude/commands/<name>.md` is a thin
  pointer. Edit `.agents/commands/<name>.md` and regenerate.

## Activation

Restart Claude Code after generating the files so it picks up `CLAUDE.md`,
`.claude/agents/`, `.claude/commands/`, and `.claude/settings.json`.

## Manual override

To customize, edit `.agents/agentic.json` and re-run
`./.agents/bootstrap.sh claude-code`. Do not edit `CLAUDE.md` or the
`.claude/agents/*.md` / `.claude/commands/*.md` files directly — they are
autogenerated and your changes will be overwritten on the next render.
