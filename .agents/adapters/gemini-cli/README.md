# gemini-cli adapter

This adapter renders the canonical manifest `.agents/agentic.json` into
Gemini CLI's native instruction format:

- `./GEMINI.md` — top-level project instructions (auto-loaded by Gemini CLI).
- `./.gemini/commands/<name>.toml` — one file per slash command in the
  manifest's `commands` array.

## How it is rendered

`./.agents/bootstrap.sh gemini-cli` (or equivalently
`./.agents/adapters/_common/render.py --cli gemini-cli --root .`) reads
`.agents/agentic.json`, applies `project_detect` rules, and writes both the
markdown and the TOML files. No LLM is involved.

## Output paths

- `./GEMINI.md` (gitignored)
- `./.gemini/commands/<name>.toml` for each command (gitignored)

To verify the rendered files are in sync with the manifest, run
`./.agents/bootstrap.sh gemini-cli --check` or `./.agents/bootstrap.sh --check`.

## Field mapping (`.agents/agentic.json` → Gemini CLI)

| `agentic.json` | Gemini output | Notes |
|---|---|---|
| `instructions: [...]` | top of `GEMINI.md` (after the framework header) | Listed as references for the model to read |
| `subagents[*]` | `### \`<name>\`` section in `GEMINI.md` | Roles are expressed as **inline instructions** in the project markdown; Gemini has no separate subagent concept with permissions |
| `subagents[*].permission` | "Permission contract" subsection | **Soft** — Gemini has no hard file ACLs. The contract is an instruction, not an enforcement |
| `commands[*]` | one `.gemini/commands/<name>.toml` | `description` and a stub `prompt` that points at `.agents/commands/<name>.md` |
| `project_detect[*]` | "Detected Stack" section in `GEMINI.md` | Surfaced as a list of stack labels |
| `project_detect[*].apply.implementer_permission_overrides` | merged into the `implementer` "Permission contract" section in `GEMINI.md` | Documented in prose rather than enforced by Gemini |

## Caveats and limitations

- **No hard permissions**: Gemini CLI cannot deny specific file edits like
  opencode can. The "Permission contract" sections are advisory.
- **No native sub-agent isolation**: a single Gemini session adopts one role at
  a time. Switching roles mid-session is possible but not enforced.
- **Slash commands are stubs**: each `.toml` points at a canonical body in
  `.agents/commands/`. Edit the canonical, regenerate the stub.

## Activation

Restart Gemini CLI after generating the files so it picks up `GEMINI.md` and
the new `.gemini/commands/` directory.

## Manual override

To customize, edit `.agents/agentic.json` and re-run
`./.agents/bootstrap.sh gemini-cli`. Do not edit `GEMINI.md` or the
`.gemini/commands/*.toml` files directly — they are autogenerated and your
changes will be overwritten on the next render.
