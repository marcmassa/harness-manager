# opencode adapter

This adapter renders the canonical manifest `.agents/agentic.json` into
`opencode.json` at the repository root, so that opencode can load the Harness
SDD framework natively.

## How it is rendered

`./.agents/bootstrap.sh opencode` (or equivalently,
`./.agents/adapters/_common/render.py --cli opencode --root .`) reads
`.agents/agentic.json`, applies `project_detect` rules, and produces
`./opencode.json` deterministically. No LLM is involved.

## Output path

`./opencode.json` at the repository root.

> This file is gitignored. Each developer runs `./.agents/bootstrap.sh opencode`
> after cloning. To verify the file is in sync with the manifest, run
> `./.agents/bootstrap.sh opencode --check` or `./.agents/bootstrap.sh --check`.

| `skills.paths: [...]` | `skills.paths: [...]` | Direct copy |
| `default_agent` | `default_agent` | Direct copy |
| `subagents[*].name` | `agent.<name>` | Object key |
| `subagents[*].mode` | `agent.<name>.mode` | Direct copy |
| `subagents[*].description` | `agent.<name>.description` | Direct copy |
| `subagents[*].role_file` | `agent.<name>.prompt` (stub) | "You are `<name>`. Your complete role is defined in `<role_file>` â€” read that file FIRST and follow it strictly." |
| `subagents[*].permission` | `agent.<name>.permission` | Direct copy. The `implementer` permission is merged with any `project_detect[*].apply.implementer_permission_overrides` |
| `commands[*].name` | `command.<name>` | Object key |
| `commands[*].description` | `command.<name>.description` | Direct copy |
| `commands[*].body_file` | `command.<name>.prompt` (stub) | "Read `<body_file>` and execute the workflow described there." |
| `project_detect[*]` | merged into `agent.implementer.permission` | Stack-aware overrides |

## Caveats and limitations

- **Stack-aware overrides**: only `implementer` is auto-overridden. If you need
  stack-aware overrides for other agents, add a `project_detect` rule in
  `.agents/agentic.json` and extend the renderer in
  `.agents/adapters/_common/render.py`.
- **opencode's `mode: "primary"` agents are limited**: only one primary agent
  can be `default_agent`. Other `mode: "primary"` entries are still declared
  but the `default_agent` is the entry point.
- **The `prompt` field is a stub**: opencode invokes the agent, the agent
  reads `.agents/subagents/<name>/SUBAGENT.md` as its first action. This is
  by design â€” it keeps `.agents/` as the single source of truth.

## Activation

After generation, restart opencode so it re-reads `opencode.json`. opencode
loads its config only at startup; it is not hot-reloaded.

## Manual override

If you need to customize an agent for a one-off, prefer editing
`.agents/agentic.json` and re-rendering, not editing `opencode.json` directly.
The generated file is marked as autogenerated; manual edits will be lost
on the next render.
